##  1. API 能实现什么功能？

API 的核心目标是：  
**让系统能够自动化地为地址补充能量 / 带宽，而不需要人工手动下单。**

### 🔧 核心能力一览

#### **1) 自动创建租赁订单**
- 支持能量 / 带宽
- 自动补能、无需人工触发

#### **2) 查询订单与资源状态**
- 查看订单是否成功
- 是否已委托到指定 TRON 地址

#### **3) 获取实时资源信息**
- 查询当前价格（能量/带宽）
- 查询配额、可用量、区间限制

#### **4) 批量处理能力**
- 批量创建订单  
- 批量查询状态  
- 适合大型机构/系统任务

#### **5) 交易能量消耗预估**
- 根据交易 HEX 预估能量消耗  
- 自动决定是否需要补能

#### **6) API Token 鉴权**
- 后端系统安全集成  
- 限流、权限、白名单控制

##  2. 适用的用户类型

Gas API 面向的是 **有自动化需求、链上操作量大** 的用户或企业。

### ✔ 特别适合以下用户

#### **• 交易所 / 托管机构**
- 大量提现、内部转账  
- 自动补能可减少失败率

#### **• DApp / 合约平台**
- 给用户提供 “Gasless” 操作体验  
- 避免用户因缺能导致交易失败

#### **• 企业级钱包 / 多地址管理系统**
- 管理成百上千的业务地址  
- 自动维护每个地址的资源水平

#### **• 自动化脚本 / Bot / 定时任务**
- 批量空投、批量 mint  
- 自动清算、自动执行任务

### ✘ 不太适合的用户

#### **• 普通个人用户**
- 建议直接在前端页面手动购买/租赁即可  
- 无需集成 API

## 🧩 3. 典型使用场景示例

### **场景 A：交易所自动补能（最常见）**
1. 用户发起提现  
2. 系统检测该地址能量不足  
3. 自动调用 API 创建订单  
4. 资源委托成功后执行提现

> **效果：** 降低失败率、减少人工介入。

---

### **场景 B：DApp Gasless 模式**
1. 用户点击执行合约  
2. 后端检测能量不足  
3. 自动补能 → 执行交易  
4. 用户无感知、不需要了解资源概念

> **效果：** 用户体验极佳。

---



## 📝 4. 小结

**GasStation API 的价值：**  
- 让你的系统具备“自动补能”能力  
- 避免因资源不足问题导致的交易失败  
- 支撑大规模、高频链上业务  
- 是交易所、DApp、钱包与任务系统的关键基础能力
